Campus Sync

Team of 3 WinForms Leadership

Overview

A WinForms application for college room booking where students and staff reserve library spaces for study and meetings with role-based access.

Role and Contribution

  • Group lead with weekly planning and communication workflows
  • UI designer for all role-specific interfaces
  • Core developer across booking and validation logic
  • Documentation owner for setup guides and progress reporting
  • Database administration across Azure, college server, and local backup

Team Collaboration

  • Team size: 3 active members
  • Tooling: GitHub, DevOps, Teams, WhatsApp, Discord
  • Weekly review meetings and feedback loop sessions

Project Glimpse

Technologies Used

  • C# WinForms
  • MySQL (Azure, college server, local)
  • DevOps and GitHub

Key Features

  • Role-based flows: Admin, Staff, Student
  • Admin-side full CRUD for users and rooms
  • Real-time booking validation and availability checks
  • Dedicated UI workflows for each user type

What I Learned

  • Prioritizing team delivery under semester constraints
  • Coordinating code quality with a smaller team
  • Managing multiple live database environments simultaneously
  • Practical leadership and real-world collaboration habits
