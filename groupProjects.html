Campus Sync

Overview

A WinForms application for college library management, allowing students and staff to book rooms for study, meetings, and other activities.

Built as a 3-person team to deliver a fully functional system with multiple user roles and real-time database integration.

Role(s) / Contribution

  • Group leader: coordinated the team, maintained communication via WhatsApp and Discord.
  • UI designer: Created UI for all user roles.
  • Programmer: Programmed core coding responsibilities across the app.
  • Documentation: Created doumentation with instructions & guidance for the team alongside with documenting the journey of the project.
  • Database admin: Database setup and management (Azure virtual DB, college server DB, local backup).

Team & Collaberations

  • Team size: 3 active members.
  • Tools: GitHub (version control), DevOps (cloud control), Teams (weekly supervisor/client meetings).
  • Weekly meetings with roleplay "supervisor" sessions for reporting progress and client feedback.

Screenshots

Technologies Used

  • C# WinForms
  • MySQL (Azure, college server, local)
  • DevOps & GitHub

Key Features

  • Three user roles: Admin, Staff, Student.
  • Admin: full CRUD of users and rooms.
  • Staff & Student: room bookings with real-time validation.
  • UI designed for each role with smooth user experience.
  • Validation to prevent invalid bookings.

What I Learned

  • Time management due to semester workload and smaller team size.
  • Setting up patching unrealiable frameworks.
  • Managing multiple live databases simultaneously.
  • Learned leadership, teamwork, real-world coding practices and UI design.
